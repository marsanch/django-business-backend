🚀 CÓMO EMPEZAR
═══════════════════════════════════════════════════════════════════════════

  OPCIÓN 1: Sin Docker (Desarrollo local)
  ─────────────────────────────────────────
  $ python -m venv venv
  $ source venv/bin/activate       # Linux/Mac
  $ pip install -r requirements.txt
  $ cp .env.example .env           # Editar valores
  $ python manage.py migrate
  $ python manage.py createsuperuser
  $ python manage.py runserver
  → Acceder a http://localhost:8000/admin/

  OPCIÓN 2: Con Docker (Production-ready)
  ────────────────────────────────────────
  $ docker-compose up -d
  $ docker-compose exec web python manage.py createsuperuser
  → Acceder a http://localhost:8000/admin/

🎯 PRÓXIMOS PASOS RECOMENDADOS
═══════════════════════════════════════════════════════════════════════════

  1. Configurar Twilio
     • Crear cuenta en twilio.com
     • Obtener Account SID y Auth Token
     • Actualizar .env

  2. Instalar y activar entorno
     • Crear venv: python -m venv venv
     • Activar: source venv/bin/activate
     • Instalar: pip install -r requirements.txt

  3. Ejecutar migraciones
     • python manage.py migrate

  4. Crear superusuario
     • python manage.py createsuperuser

  5. Iniciar desarrollo
     • python manage.py runserver
     • O: docker-compose up -d

  6. Probar API
     • GET http://localhost:8000/api/appointments/businesses/
     • Ver API_DOCS.md para ejemplos
